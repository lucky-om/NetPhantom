User Guide & Reference

1. Getting Started

NetPhantom captures raw frames traversing local Network Interface Cards (NICs). To operate on Windows or Linux, kernel-level promiscuous socket driver wrappers must be configured.

Setup Target

Zero-Config CLI Entry Point

The Setup Wizard links executable files and configures the system PATH so you can launch NetPhantom anywhere by typing netphantom in terminal.

Windows Hardware Driver Setup

Ensure Npcap is installed with "Install Npcap in WinPcap API-compatible mode" enabled. Without this driver, Scapy cannot list hardware interfaces.

3. BPF Capture Syntax Table

Filtering traffic at the capture layer reduces processing overhead on sniffer threads:

BPF Syntax Target Traffic & Description
tcp Interprets only TCP socket connection frames.
udp port 53 Filters DNS name resolution queries traversing port 53.
host 192.168.1.1 Captures packets where source or destination matches host IP.
src port 80 or dst port 443 Filters HTTP or TLS web traffic sessions.
icmp Captures network pings and ICMP control messages.

4. Keyboard Shortcuts

Control capture states and navigate the dashboard with high-speed keyboard shortcuts:

Key Trigger Action Description
F5 Start live packet capture on selected interface.
F6 / Escape Stop active packet capture session.
Ctrl + R Restart capture (flushes buffer and restarts).
Ctrl + O Open and import a local .pcap capture trace file.
Ctrl + S Save captured packets to a PCAP or JSON log file.
Ctrl + L Clear packet list and reset throughput metrics.

5. PCAP Session Import & Export

All active captures can be exported directly to disk in standard .pcap format compatible with Wireshark, tcpdump, and Snort. You can also re-open saved capture logs to audit security events offline.

Getting Started

NetPhantom captures raw packets traversing network interface cards (NICs). To run successfully on Windows or Linux, the underlying execution environment must load specialized kernel-level packet driver wrappers.

Zero-Config CLI Entry Point

The Graphical Setup Installer automatically copies required source scripts and runs pip links to configure the global netphantom command line shortcut on your system.

Windows Driver Configuration

Ensure Npcap is installed with the option "Install Npcap in WinPcap API-compatible mode" checked. Without this driver, Scapy will fail to list interfaces or bind to interfaces.

BPF Capture Filters

NetPhantom supports Berkeley Packet Filter (BPF) syntax at the capture layer. Filtering at capture time reduces the processing overhead on the sniffer threads.

Common filter templates you can type in the Capture Filter input field:

Filter Syntax Description
tcp Intercepts only TCP socket connections.
udp port 53 Filters for DNS traffic traversing port 53.
host 192.168.1.1 Captures packets where source or destination IP matches 192.168.1.1.
src port 80 or dst port 443 Filters for outgoing HTTP or TLS web browsing packets.
icmp Captures network pings and control message protocols.

Keyboard Shortcuts

Manage capture states and navigate the dashboard quickly using terminal-style keyboard triggers.

Key Trigger Action
F5 Start capture on selected interface.
F6 / Escape Stop current capture session.
Ctrl + R Restart capture (stops, clears, and starts).
Ctrl + O Open/Import local .pcap capture file.
Ctrl + S / Ctrl + E Save currently captured packets to a .pcap file.
Ctrl + L Clear packet list and reset stats counters.
Ctrl + F Focus display search filter input field.
Ctrl + Plus / Ctrl + Minus Zoom font sizes in packet list.

PCAP Import & Export

NetPhantom fully supports the standard `.pcap` capture format. This allows seamless integration with other tools like Wireshark, TShark, or tcpdump.

Dumping Sessions

Post-Capture Log Audits

When you finish sniffing, click File → Save as PCAP. You can load this file directly back into NetPhantom or Wireshark later. For developer scripts, you can export the parsed packet summaries as JSON using File → Export as JSON.
